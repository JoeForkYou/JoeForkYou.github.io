回调函数详解

1. 什么是回调函数？

回调函数（Callback Function）是一个通过函数指针调用的函数。它允许将函数作为参数传递给另一个函数，实现了一种松耦合的编程方式。

C语言中的设计模式

设计模式概述

设计模式是软件开发中常见问题的典型解决方案。在C语言中，尽管没有面向对象的特性，我们仍然可以通过结构体和函数指针来实现这些模式。以下是最常用的几种设计模式及其实现。

graph TB A[C语言设计模式] --> B[创建型模式] A --> C[结构型模式] A --> D[行为型模式] B --> B1[单例模式] B --> B2[工厂模式] B --> B3[建造者模式] C --> C1[适配器模式] C --> C2[组合模式] C --> C3[代理模式] D --> D1[观察者模式] D --> D2[策略模式] D --> D3[命令模式]

1. 单例模式（Singleton Pattern）

单例模式确保一个类只有一个实例，并提供一个全局访问点。

图解单例模式

sequenceDiagram participant Client as 客户端 participant Singleton as 单例对象 participant Instance as 实例存储 Client->>Singleton: 1. 请求实例 get_instance() Singleton->>Instance: 2. 检查实例是否存在 alt 实例不存在 Instance->>Singleton: 2.1 返回 NULL Singleton->>Singleton: 2.2 创建新实例 Singleton->>Instance: 2.3 存储实例 else 实例存在 Instance->>Singleton: 2.1 返回现有实例 end Singleton->>Client: 3. 返回实例

2. 观察者模式（Observer Pattern）

观察者模式定义了对象之间的一对多依赖关系，使得当一个对象改变状态时，所有依赖于它的对象都会得到通知并自动更新。

图解观察者模式

classDiagram class Subject { +observers: Observer[] +attach(Observer) +detach(Observer) +notify() } class Observer { +update() } class ConcreteObserver { +update() } Subject --> Observer Observer <|-- ConcreteObserver
sequenceDiagram participant S as Subject participant O1 as Observer1 participant O2 as Observer2 Note over S,O2: 注册观察者 S->>O1: attach(observer1) S->>O2: attach(observer2) Note over S,O2: 状态改变 S->>S: 状态更新 S->>O1: notify() S->>O2: notify() O1->>O1: update() O2->>O2: update()

3. 策略模式（Strategy Pattern）

策略模式定义了一系列算法，并使这些算法可以相互替换。策略模式使算法可以独立于使用它的客户端而变化。

图解策略模式

classDiagram class Context{ +Strategy strategy +execute() } class Strategy{ +algorithm() } class ConcreteStrategyA{ +algorithm() } class ConcreteStrategyB{ +algorithm() } Context --> Strategy Strategy <|.. ConcreteStrategyA Strategy <|.. ConcreteStrategyB
graph TD A[客户端] --> B[上下文Context] B --> C{策略选择} C -->|策略A| D[加法策略] C -->|策略B| E[减法策略] C -->|策略C| F[乘法策略] D --> G[执行策略] E --> G F --> G
